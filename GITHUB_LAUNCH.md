## Release summary

WebWatcher v0.3.0 turns the project into a much more practical self-hosted monitoring base.

It now supports:
- CSS selector-based monitoring
- noise filtering for unstable content
- Playwright for JavaScript-rendered pages
- JSON config file support
- Feishu webhook notifications
- Docker / docker-compose

This release is still intentionally small:
- no user system
- no big dashboard
- no SaaS complexity

Just a hackable, local-first website change monitor that is easy to understand and easy to extend.

## Short launch post (EN)

Just wrapped up **WebWatcher v0.3.0**.

It is a small open-source project with a very specific goal:
**watch webpage changes without building a huge monitoring platform.**

Current features:
- webpage change detection
- CSS selector-based monitoring
- noise filtering for unstable content
- Playwright support for JavaScript-rendered pages
- JSON config file support
- SQLite local event history
- Feishu webhook notifications
- Docker / docker-compose support

I intentionally kept it small:
- no SaaS dashboard
- no user system
- no overbuilt architecture

If you want a simple, hackable, self-hosted starting point for website monitoring, this version is ready.

Good fit for:
- job pages
- announcement pages
- documentation updates
- rankings / listings
- competitor pages
- simple dynamic pages
